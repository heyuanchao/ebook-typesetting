# 电子书排版工作流

适用于将 `.docx` 电子书整理为适合阅读/打印的版本。下次可直接对 Claude 说：

```text
按《电子书排版工作流.md》处理 XXX.docx
```

---

## 1. 准备与备份

1. 明确输入文件：
   - 原始全文版，例如：`书名_全文.docx`
   - 工作排版版，例如：`书名_全文_重新排版.docx`
2. 每次做大改前先备份，不直接裸改：

```text
书名_处理项前备份.docx
```

示例：

```text
书名_封面调整前备份.docx
书名_错别字修正前备份.docx
书名_样式统一前备份.docx
书名_标题分页前备份.docx
```

---

## 2. 封面处理

### 目标

首页只放：

```text
书名
作者
```

### 推荐样式

- 书名：黑体 / 加粗 / 大字号 / 深蓝或黑色
- 作者：楷体或宋体 / 小字号 / 灰色或黑色
- 书名、作者水平居中
- 首页内容上下居中
- 目录从下一页开始

### 检查点

确认：

```text
封面单独一页
目录从第二页开始
书名和作者水平居中
封面内容视觉上下居中
```

---

## 3. 目录处理

### 目标

目录清晰、层级明显、可跳转。

### 普通目录项

普通目录项统一样式，例如：

```text
蓝色 / 左对齐 / 可点击
```

### PART 类目录项

PART 类目录项要突出显示：

```text
黑体 / 加粗 / 深蓝色 / 稍大字号 / 无下划线
```

例如：

```text
PART1 主动选择，才能接近梦想
PART2 做从未来看正确的事
```

### 检查点

确认：

```text
目录项数量正确
目录跳转目标存在
PART 类标题比普通标题突出
无无效链接
无跳转失败项
```

---

## 4. 正文标题层级

### 一级标题

用于：

```text
推荐序
自序
PART
附录
```

样式建议：

```text
黑体 / 加粗 / 大字号 / 深蓝
段前较大
段后适中
必要时分页
```

### 普通文章标题

用于每篇文章标题，例如：

```text
不叛逆，就不曾拥有青春
快乐起来，找到自己的城市
我为什么从不找一份稳定的工作
```

样式建议：

```text
黑体 / 加粗 / 蓝色 / 中字号
段前适中
段后适中
```

### 重要规则

普通文章标题 **不要全部强制新页**。  
否则打印时会出现大量大面积空白页。

建议：

```text
PART 页可以单独成页
推荐序/自序/附录可以新页开始
普通文章标题不强制分页
```

---

## 5. PART 页处理

### 推荐方案

PART 页单独成页，并视觉上下居中：

```text
PART1 主动选择，才能接近梦想
```

样式：

```text
黑体 / 加粗 / 深蓝 / 大字号
水平居中
视觉上下居中
单独分页
```

### 注意

只对 PART 页上下居中，不要对所有普通文章标题上下居中。

---

## 6. 正文段落样式统一

### 推荐正文样式

```text
字体：宋体
字号：12号
颜色：#222222 或黑色
首行缩进：2字符
对齐：两端对齐
段前：0
段后：适中
行距：固定值或多倍行距，保持全书一致
```

### 检查点

确认：

```text
正文段落字体一致
字号一致
颜色一致
首行缩进一致
行距一致
没有某些段落变成默认字体
```

---

## 7. 作者署名行

例如：

```text
◎冯仑
◎同道大叔
◎鲁豫
◎孙宇晨
```

### 推荐样式

```text
黑体 / 加粗 / 蓝色
段前较大
段后适中
不进入目录层级
不出现在导航目录中
```

### 检查点

确认：

```text
作者署名不出现在目录中
作者署名样式全书一致
```

---

## 8. 引语 / 小标题

例如：

```text
这个世界不会亏待对自己负责的人
田野与树木没有给我一点教益，而城市的人们却赐给我颇多的教益。——苏格拉底
```

### 推荐样式

```text
黑体 / 加粗 / 蓝色
居中
段前段后适中
不进入正式目录层级
```

### 检查点

确认：

```text
引语样式一致
不会误进目录
不会影响正文导航
```

---

## 9. 错别字与标点检查

### 检查内容

重点检查：

```text
中文左右引号
英文半角引号
中英文括号混用
英文逗号/句号误用
明显 OCR 错字
语序异常
标点粘连
```

### 常见问题示例

```text
”90后”      → “90后”
”80后”      → “80后”
CEO,CEO     → CEO，CEO
All in.某   → All in。某
XRP(Ripple） → XRP（Ripple）
范筹        → 范畴
沉弱        → 沉溺
行同路人    → 形同路人
```

### 不确定项规则

对于无法确定的句子，不要擅自改，先列出来让用户确认。

例如：

```text
在华尔街路过演
堪比一个人人生的战争祭祀
```

这类只标记为“可疑”，不自动修改。

---

## 10. 与原文对比

### 目标

确认重新排版版没有误删、误改正文。

### 对比范围

只对比正文，不把这些算入正文差异：

```text
封面
目录
尾部附加目录
格式
分页
样式
```

### 检查项

```text
正文段落数
正文字符数
正文非空白字符数
正文哈希
逐段差异
```

### 差异分类

将差异分为：

1. 已确认错别字修正
2. 标点规范化
3. 段落拆分/合并
4. 未确认的正文变化

只有第 4 类需要重点复核。

---

## 11. 空白页检查

### 常见原因

```text
每个普通标题都设置了段前分页
空段落里带分页符
PART 页上下居中过度
标题前后间距过大
分节符导致空白页
```

### 推荐策略

保留分页：

```text
封面后
目录后
推荐序/自序
每个 PART
附录
```

取消分页：

```text
普通文章标题
普通正文小标题
引语
作者署名
```

### 经验规则

如果看到很多空白页，优先检查普通文章标题是否被全部设置了“段前分页”。

---

## 12. 打印前最终检查

打印前按顺序确认：

```text
1. 封面是否单独一页
2. 目录是否清晰
3. PART 是否突出
4. 目录点击是否可跳转
5. 正文段落样式是否统一
6. 普通标题样式是否统一
7. 作者署名是否未进入目录
8. 是否还有异常标点
9. 是否还有大面积空白页
10. 与原文正文是否只有确认过的差异
```

---

## 13. 推荐最终排版规则汇总

```text
封面：上下居中
目录：普通目录项统一，PART 项突出
推荐序/自序：新页开始
PART：单独页，视觉上下居中
普通文章标题：不强制新页，避免空白页
正文：宋体 12号，两端对齐，首行缩进
作者署名：不进目录
引语：居中，不进目录
打印前：检查错别字、标点、样式、空白页、正文差异
```

---

## 14. 下次使用提示词

可直接复制：

```text
请按《电子书排版工作流.md》处理 XXX.docx。
要求：先备份；按流程处理封面、目录、PART、标题、正文、作者署名、引语、错别字和标点；修复目录跳转；删除多余空白页；最后检查样式统一和正文差异，并列出修改内容和备份文件名。
```
